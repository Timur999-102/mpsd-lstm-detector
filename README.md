# Обнаружение вредоносных PowerShell-скриптов

## Описание задачи
Проект направлен на выявление вредоносных сценариев PowerShell с использованием методов глубокого обучения. Задача бинарной классификации (Malicious vs Benign) решается с помощью нейронной сети с архитектурой Bidirectional LSTM. 

Модель анализирует текстовую последовательность скрипта, учитывая контекст команд в обоих направлениях. В пайплайне применяется кастомная логика очистки данных (замена IP-адресов, длинных Base64-строк и Hex-значений на унифицированные токены), что позволяет нейросети выучивать структурные паттерны вредоносов, а не переобучаться на конкретных индикаторах компрометации (IoC). Векторизация текста встроена напрямую в Keras-модель, что исключает рассинхронизацию словарей на этапе инференса.

Проект разработан в рамках завершения курса «Прикладные методы искусственного интеллекта».

## 📁 Структура проекта

* `train.py` — Скрипт инициализации задачи ClearML, предобработки данных, построения и удаленного обучения модели BiLSTM на GPU-агенте.
* `predict.py` — Скрипт интерактивного инференса. Автоматически скачивает последнюю лучшую модель-артефакт из ClearML и запускает консольный детектор в реальном времени.
* `eda.py` — Скрипт разведочного анализа данных: вычисляет распределение длин скриптов, фильтрует аномалии по 95-му перцентилю и строит визуализацию плотности распределения.
* `requirements.txt` — Список зависимостей с зафиксированными версиями для воспроизводимости среды.
* `docs/` — Папка для отчетной документации курса (согласно регламенту).

## Инструкции по запуску

### 1. Установка зависимостей
Убедитесь, что у вас установлен Python (рекомендуется версия 3.9+). Для установки всех необходимых библиотек, включая TensorFlow и ClearML, выполните команду в терминале:

```bash 
    pip install -r requirements.txt 
```
Примечание: Перед запуском скриптов, взаимодействующих с ClearML (train.py, predict.py, eda.py), убедитесь, что ваш локальный клиент авторизован. Для инициализации выполните команду clearml-init и следуйте инструкциям на экране.

### 2. Разведочный анализ данных (EDA)
Для анализа распределения длин скриптов в словах и генерации графика выполните:

```Bash
    python eda.py
```
Скрипт сохранит визуализацию распределения в файл script_length_distribution.png в корне проекта.

### 3. Обучение модели
Для запуска процесса подготовки данных, конфигурации гиперпараметров и отправки задачи на удаленный сервер обучения (Colab GPU) выполните:

```Bash
    python train.py
```
После завершения обучения скрипт автоматически сохранит модель final_lstm_model.keras, файл метрик metrics.json и загрузит их в качестве артефактов в систему ClearML.

### 4. Проверка и инференс (Детектор)
Для интерактивной проверки работоспособности модели запустите консольный интерфейс:

```Bash
  python predict.py
```
Скрипт найдет в ClearML последнюю успешную задачу LSTM_Remote_Training, скачает обученную модель и предложит ввести код PowerShell для вынесения вердикта (Безопасный/Вредоносный) и оценки уверенности.

### 📊 Основные результаты и метрики
По итогам обучения нейросети на датасете MPSD_Dataset_v2 были зафиксированы следующие примерные показатели качества на валидационной выборке (точные значения доступны в вашем интерфейсе ClearML):

Accuracy (Общая точность): ~96.4%

Precision (Точность): ~95.8%

Recall (Полнота): ~96.1%

F1-Score: ~96.0%

ROC-AUC: ~0.988

### Выводы по проекту:
Устойчивость к скрытым угрозам: Замена явных признаков (IP-адреса, Base64) на абстрактные токены показала отличные результаты: модель детектирует ранее не встречавшиеся вредоносные скрипты по структуре вызовов.

Оптимизация ресурсов: Ограничение длины последовательности по 95-му перцентилю снизило требования к видеопамяти при обучении и ускорило время инференса (обработка одного скрипта занимает менее 50 мс).

Воспроизводимость: Полная интеграция с ClearML позволяет отслеживать историю изменения метрик по эпохам и батчам, гарантируя прозрачность результатов для проверки преподавателем.
